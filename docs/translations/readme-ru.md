# Cyanide

**Cyanide** — это SSH & Telnet ханипот (honeypot) с высоким уровнем взаимодействия, предназначенный для обмана и анализа поведения злоумышленников. Он объединяет реалистичную эмуляцию файловой системы Linux, продвинутую симуляцию команд и ML-движок для обнаружения аномалий.

---

## 🌟 Ключевые Возможности

### 🧠 Реалистичная Эмуляция
*   **Мульти-протокол**: Поддержка SSH (`asyncssh`) и Telnet на разных портах (по умолчанию 2222/2223).
*   **Динамическая VFS**: Полнофункциональная файловая система в памяти, загружаемая из YAML-профилей. Изменения сохраняются в рамках сессии.
*   **Продвинутый Shell**: Поддержка пайпов (`|`), перенаправлений (`>`, `>>`), логических операторов (`&&`, `||`) и переменных окружения.
*   **Anti-Fingerprinting**:
    *   **Сетевой Джиттер**: Случайные задержки ответов (50-300мс).
    *   **OS Профили**: Реалистичная маскировка под **Ubuntu**, **Debian** или **CentOS** с динамическими списками процессов (`ps`), файлами `/proc` и историческими таймстампами ФС.

### 🛡️ Гибридная Система Обнаружения
Cyanide использует трехуровневый движок для выявления угроз:
1.  **ML Детектор Аномалий**: Нейросеть (автоэнкодер) выявляет нестандартные структуры команд (0-day, обфускация).
2.  **Движок Правил Безопасности**: Regex-сигнатуры для известных атак (`wget`, `curl | bash` и т.д.).
3.  **Контекстный Анализатор**: Семантический анализ доступа к файлам (`/etc/shadow`) и проверка репутации по GeoIP.

### 🚀 Оркестрация VM (Backend Pool)
*   **VMPool**: Динамическое управление пулом бэкенд-виртуалок через **Libvirt/KVM**.
*   **Изоляция**: На каждую сессию создается чистый клон VM, который уничтожается сразу после отключения хакера.
*   **Прозрачный Proxy**: Перехват и расшифровка SSH/Telnet трафика в режиме MiTM (Man-in-the-Middle).

### 🕵️ Расширенные функции (Advanced)
*   **Honeytokens**: Встроенные файлы-ловушки (`/etc/shadow`, `id_rsa`). Любое касание этих файлов вызывает **CRITICAL_ALERT**.
*   **HTTP Log Browser**: Встроенный веб-интерфейс для просмотра логов и TTY-записей через браузер (порт 9090).
*   **SMTP Proxy**: Перехват и анализ исходящего спама или попыток эксфильтрации данных через почту.
*   **Устойчивая Мимикрия**: Сохранение SSH-ключей хоста между перезапусками, чтобы отпечаток (fingerprint) сервера не менялся.

## 🚀 Развертывание

**Рекомендуется запускать через Docker.**

### 🐳 Docker Compose

```bash
# 1. Запуск полного стека (Ханипот + MailHog + Jaeger)
docker-compose -f deployments/docker/docker-compose.yml up --build -d

# 2. Просмотр логов
docker-compose -f deployments/docker/docker-compose.yml logs -f cyanide

# 3. Остановка
docker-compose -f deployments/docker/docker-compose.yml down
```

### 🔧 Конфигурация

Настройки управляются через **YAML** файлы в `configs/`:

| Файл | Назначение |
|------|------------|
| `configs/app.yaml` | Основная конфигурация (порты, таймауты, настройки ML). |
| `configs/profiles/*.yaml` | Профили ОС. Определяют структуру файловой системы и метаданные. |
| `configs/fs.yaml` | (Опционально) Пользовательский шаблон ФС, если нужно переопределить случайный выбор. |

**Переменные окружения** в `docker-compose.yml` имеют приоритет над настройками `app.yaml`.

## 🛠️ Управление и Инструменты

Скрипты в `scripts/management/` помогают управлять ханипотом:

| Скрипт | Команда | Описание |
|--------|---------|----------|
| **Stats** | `python3 scripts/management/stats.py` | Показать статистику в реальном времени (аптайм, сессии). |
| **Replay** | `scriptreplay <timing> <log>` | Воспроизвести TTY сессию (файлы в `var/log/cyanide/tty/`). |

## 🕵️ Данные и Форензика

*   **Логи**: `var/log/cyanide/`
    *   `cyanide-server.json`: Системные события, жизненный цикл сервисов и ошибки.
    *   `cyanide-fs.json`: Действия злоумышленника (команды, попытки авторизации, TTY сессии).
    *   `cyanide-ml.json`: Результаты работы ML и детальные "мысли" системы.
    *   `cyanide-stats.json`: Периодические срезы статистики ханипота.
    *   `tty/<session_id>/`: Файлы полной записи TTY сессий.
*   **Карантин**: `var/quarantine/`
    *   Скачанные и загруженные файлы.
